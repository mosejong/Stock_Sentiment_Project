# Gemini 기반 실시간 주식 분석 대시보드

뉴스, 차트 데이터, 과거 예측 로그를 결합해 종목별 방향성을 분석하는 Streamlit 기반 AI 주식 분석 프로젝트입니다.

> 본 프로젝트는 투자 추천 서비스가 아니라, 뉴스와 주가 데이터를 정리하고 해석하는 **AI 기반 분석 보조 도구**입니다.

이 프로젝트는 **Google Gemini AI**와 **실시간 Google Search 기능**을 활용해 매일 종목별 뉴스 재료, 기술적 지표, 과거 패턴을 함께 분석합니다.  
분석 결과는 CSV로 누적 저장되며, Streamlit 대시보드에서 종목별 예측 방향과 근거를 확인할 수 있습니다.

---

## 핵심 기능

- 종목별 최신 뉴스 수집 및 Gemini 기반 요약
- 종가, 이동평균선(MA20, MA60), 거래량 기반 차트 흐름 분석
- 과거 예측 로그를 활용한 패턴 참고
- AI 예측 결과를 CSV 파일로 누적 저장
- 과거 날짜 기준 백필 데이터 생성
- Streamlit 대시보드로 종목별 분석 결과 시각화
- 향후 적중률 평가 및 ML 모델 학습을 위한 데이터 구조 준비

## Quick Start

### 1. 환경 변수 설정

프로젝트 루트 폴더에 `.env` 파일을 만들고 Google API 키를 입력합니다.

```env
GOOGLE_API_KEY=your_google_api_key_here
```

### 2. 전체 파이프라인 실행

```powershell
# 가상환경 활성화 (Windows)
.\.venv\Scripts\Activate.ps1

# 필수 패키지 설치
pip install -r requirements.txt

# 데이터 업데이트 → AI 분석 → 성과 평가 → 대시보드 실행
python run_pipeline.py
```

### 3. 개별 실행

```bash
# 주가 데이터 업데이트
python src/update_data.py

# 오늘 기준 AI 분석 리포트 생성
python src/main_auto.py

# 과거 데이터 백필
python src/backfill_history.py

# 예측 결과 평가 / 통합
python src/evaluator.py

# 대시보드 실행
streamlit run src/web_app.py
```

## 분석 파이프라인

이 프로젝트는 상위 실행 파일인 `run_pipeline.py`를 통해 아래 흐름을 자동으로 연결합니다.

### 1단계. 주가 데이터 업데이트

`src/update_data.py`가 종목별 원본 차트 데이터를 갱신합니다.

### 2단계. AI 분석 리포트 생성

`src/main_auto.py`가 당일 뉴스, 차트 흐름, 과거 패턴을 종합해 종목별 AI 분석 결과를 생성합니다.

### 3단계. 예측 결과 평가 / 통합

`src/evaluator.py`가 누적된 예측 결과를 평가하고 통합 성과 파일을 갱신합니다.

즉, `run_pipeline.py`는 **데이터 갱신 → AI 분석 → 성과 평가 → 대시보드 실행**을 하나의 흐름으로 묶는 상위 파이프라인입니다.

## 현재 분석 구조

### 뉴스 분석

- Google Search Tool을 활용해 종목 관련 최신 뉴스를 수집합니다.
- Gemini가 핵심 키워드, 뉴스 요약, 뉴스 성격을 정리합니다.

### 차트 분석

- 종가, 이동평균선, 거래량 등 기술적 지표를 기반으로 현재 흐름을 해석합니다.
- 차트 흐름은 강세 / 약세 / 혼조 형태로 반영됩니다.

### 과거 패턴 참고

- 기존 예측 로그를 누적 저장하고, 종목별 과거 예측 성향과 적중 여부를 참고할 수 있도록 설계했습니다.
- 이를 통해 단순한 하루치 분석이 아니라 누적 맥락 기반 판단으로 확장할 수 있습니다.

### 최종 LLM 판단

Gemini는 뉴스, 차트, 과거 패턴 정보를 종합해 다음 항목을 생성합니다.

- AI예측
- 확신도
- 핫키워드
- 뉴스판정
- 뉴스요약
- 뉴스출처
- 패턴판정
- 차트판정
- 핵심사유

## 현재 구현된 기능

- 당일 기준 최신 뉴스와 차트 데이터 기반 종목별 분석
- 분석 결과를 `logs/daily_analysis_report.csv`에 누적 저장
- 과거 날짜 기준 백필 분석 결과 생성
- 거래일 기준 백필 실행 및 주말 자동 스킵
- Streamlit 기반 대시보드 화면 제공
- `evaluator.py`를 통한 예측 결과 평가 및 `logs/total_performance.csv` 생성

현재 단계에서는 직접 학습한 머신러닝 모델이 예측하는 구조가 아니라,  
Gemini가 뉴스와 차트 데이터를 해석할 수 있도록 데이터 파이프라인과 분석 구조를 설계한 상태입니다.

## 확신도 기준

현재 버전에서는 뉴스, 차트, 과거 패턴을 종합해 확신도를 산정합니다.

- **85~95**: 차트 / 뉴스 / 과거 패턴이 강하게 일치
- **70~84**: 우세 방향은 있으나 일부 불확실성 존재
- **55~69**: 혼조세, 관망에 가까운 구간
- **0~54**: 근거 부족 또는 신뢰 낮음

현재 구조상 5년치 차트 데이터가 항상 존재하기 때문에,  
실제 운영에서는 `0~54` 구간보다 `55~69` 구간이 보수적 판단 영역으로 더 자주 사용됩니다.

## 뉴스 반영 방식과 한계

현재 버전에서 뉴스 중요도는 명시적인 수치 가중치 방식이 아니라,  
Gemini가 핵심 뉴스를 요약하고 판단하는 과정에서 암묵적으로 반영되는 구조입니다.

즉, 특정 뉴스가 다른 뉴스보다 훨씬 중요하더라도 현재 구조에서는 LLM이 이를 자연어적으로 우선 반영하는 수준이며,  
별도의 숫자 기반 중요도 점수는 아직 적용하지 않았습니다.

## 데이터 및 보안

- API 키는 `.env` 파일에 저장하며 Git에 포함하지 않습니다.
- 분석 결과 CSV와 원본 주가 데이터는 `logs/` 하위에 저장됩니다.
- 로그 데이터는 실행 환경에 따라 생성되므로 저장소에는 포함하지 않습니다.
